## 4. 比較方法・正規化
- 比較方法: **多重集合**（行の出現回数つき集合差。重複行があるため set ではなく Counter で突合）
- **Snowflake 側を BigQuery に揃えてある**ため、比較時の“ごまかし正規化”（trim / `#N/A`→NULL / 過度な丸め）は**行わない**。
- 行うのは **DB 間の型表現差の吸収のみ**:
  - 数値: `float` に統一（Decimal/Int/Float の差を吸収。IEEE 表現ゆらぎ対策で小数9桁丸め）
  - 日付: `YYYY-MM-DD`（DATE 同士。時刻は持たない）
  - 真偽: bool
  - 文字: **そのまま**（trim しない・`#N/A` 等もそのまま比較・空文字と NULL は両DBで NULL）
- 列名は小文字化して突合（値の大小文字は変えない）。
- 比較から外す列（masking ではなく「比較しない列」を明示。理由を残す）:
  - 例: `source_sheet_name`（SF=正規化 yyyymm `202510` / BQ=生シート名 `2025.1` の**設計差**。データ本体は同一）
